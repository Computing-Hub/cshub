Example 1: Storing Questions

The first thing any quiz game needs is questions. But how do you store a question and its answer together? A dictionary is perfect for this — each dictionary holds a "question" and an "answer" as a pair. We then put all of our question dictionaries into a list, giving us a neat collection we can loop through later.

Python Code 
# Storing quiz questions as a list of dictionaries
questions = [
    {"question": "What is the capital of France?", "answer": "paris"},
    {"question": "What is 7 * 8?", "answer": "56"},
    {"question": "What colour do you get mixing red and blue?", "answer": "purple"},
    {"question": "How many sides does a hexagon have?", "answer": "6"},
]

# Let's check our data structure
print(f"Number of questions: {len(questions)}")
print()

# We can access individual questions like this
print("First question:", questions[0]["question"])
print("Its answer:", questions[0]["answer"])
print()

# And loop through them all
for i, q in enumerate(questions, 1):
    print(f"Q{i}: {q['question']}")

Example 2: The Quiz Loop

Now that we have our questions stored, we need a loop that presents each question, checks the answer, and keeps track of the score. Since input() does not work in the browser, we will simulate the user’s answers using a list. In a real Python program running on your computer, you would use input() to get the player’s actual response.

Python Code 
# Our quiz questions
questions = [
    {"question": "What is the capital of France?", "answer": "paris"},
    {"question": "What is 7 * 8?", "answer": "56"},
    {"question": "What colour do you get mixing red and blue?", "answer": "purple"},
    {"question": "How many sides does a hexagon have?", "answer": "6"},
]

# Simulated user answers (since input() doesn't work in the browser)
# In a real program you'd write: user_answer = input("Your answer: ")
simulated_answers = ["paris", "54", "purple", "6"]

score = 0

for i, q in enumerate(questions):
    print(f"Question {i + 1}: {q['question']}")

    # In a real program: user_answer = input("Your answer: ")
    user_answer = simulated_answers[i]  # simulated answer
    print(f"Your answer: {user_answer}")

    if user_answer.lower() == q["answer"]:
        print("Correct!")
        score += 1
    else:
        print(f"Wrong! The answer was {q['answer']}")
    print()

print(f"Final Score: {score}/{len(questions)}")

Example 3: Adding Functions

Our quiz works, but all the code is running in one big block. Real programs are organised into functions — each function handles one job. This makes the code easier to read, easier to test, and easier to extend. Let’s refactor our quiz into three clear functions: one to create the questions, one to run the quiz, and one to show the results.

Python Code 
def create_questions():
    """Return a list of quiz question dictionaries."""
    questions = [
        {"question": "What is the capital of France?", "answer": "paris"},
        {"question": "What is 7 * 8?", "answer": "56"},
        {"question": "What colour do you get mixing red and blue?", "answer": "purple"},
        {"question": "How many sides does a hexagon have?", "answer": "6"},
        {"question": "What planet is closest to the Sun?", "answer": "mercury"},
    ]
    return questions


def run_quiz(questions, player_answers):
    """Run through the quiz and return the score."""
    score = 0
    for i, q in enumerate(questions):
        print(f"\nQuestion {i + 1}: {q['question']}")

        # In a real program: answer = input("Your answer: ")
        answer = player_answers[i]
        print(f"Your answer: {answer}")

        if answer.lower() == q["answer"]:
            print("Correct!")
            score += 1
        else:
            print(f"Wrong! The answer was {q['answer']}")
    return score


def show_results(score, total):
    """Display the final results with a rating."""
    print("\n" + "=" * 30)
    print(f"  FINAL SCORE: {score}/{total}")
    print("=" * 30)

    percentage = (score / total) * 100
    if percentage == 100:
        print("  Perfect score! Outstanding!")
    elif percentage >= 70:
        print("  Great job! Well done!")
    elif percentage >= 50:
        print("  Not bad! Keep practising!")
    else:
        print("  Keep learning, you'll get there!")


# --- Main program ---
print("Welcome to the Python Quiz!")
print("=" * 30)

questions = create_questions()
simulated_answers = ["paris", "56", "purple", "6", "mercury"]
score = run_quiz(questions, simulated_answers)
show_results(score, len(questions))

How Does It All Fit Together?

Planning a Project

Before you write a single line of code, it helps to plan. Professional programmers spend a significant amount of time planning before they start coding. For our quiz game, we can break the project into clear parts:

  1. Store the data — What questions will the quiz ask? How will we store them?
  2. Run the quiz — How will we present questions and check answers?
  3. Track the score — How will we count correct answers?
  4. Show results — How will we display the final score and give feedback?

Each of these parts can become a function. Each function has one job. When you combine them, you get a complete program. This approach — breaking a big problem into smaller pieces — is called decomposition, and it is one of the most important skills in computer science.

Every Concept in One Program

Look at how many topics from this course appear in our quiz game:

Concept How It’s Used in the Quiz
Dictionaries Each question is stored as a dictionary with "question" and "answer" keys
Lists All the question dictionaries are collected in a list
Loops A for loop iterates through each question
Conditions if/elif/else checks whether the answer is correct and assigns a rating
Functions The code is organised into functions like create_questions(), run_quiz(), and show_results()
Strings .lower() is used to compare answers without worrying about capitals
Variables score, percentage, and total track the player’s progress
Error handling Can be added to handle unexpected input gracefully with try/except
Key Concept: Real programs are never about just one concept. They combine many ideas — variables, loops, conditions, functions, data structures — working together. The skill of programming is not just knowing each concept individually, but knowing how to weave them together to solve a problem.

Ideas for Making It Your Own

Once your basic quiz works, the fun really begins. Here are some ideas for extending your game:

  • Categories — Add a "category" key to each question dictionary (e.g., "science", "history", "maths") and let the player choose a category
  • Difficulty levels — Add a "difficulty" key with values like "easy", "medium", or "hard", and award more points for harder questions
  • Random question order — Use import random and random.shuffle(questions) to mix up the order each time
  • High scores — Keep track of the best score across multiple rounds
  • Timed questions — In a desktop Python program, you could add a time limit for each question
  • Multiple choice — Add an "options" key with a list of possible answers, making it a multiple-choice quiz
Python Code 
# Example: Adding categories and difficulty to your questions
import random

questions = [
    {"question": "What is H2O?", "answer": "water", "category": "science", "difficulty": "easy"},
    {"question": "Who wrote Romeo and Juliet?", "answer": "shakespeare", "category": "english", "difficulty": "easy"},
    {"question": "What is the square root of 144?", "answer": "12", "category": "maths", "difficulty": "medium"},
    {"question": "What gas do plants absorb?", "answer": "carbon dioxide", "category": "science", "difficulty": "medium"},
    {"question": "In what year did World War 2 end?", "answer": "1945", "category": "history", "difficulty": "hard"},
]

# Shuffle the questions for variety
random.shuffle(questions)

# Display questions with their metadata
for i, q in enumerate(questions, 1):
    print(f"Q{i} [{q['category'].upper()} - {q['difficulty']}]: {q['question']}")
    print(f"   Answer: {q['answer']}")
    print()
(click "Run Code" to see output)
Watch out: When you extend your program, add one feature at a time and test it before moving on. If you try to add everything at once, it becomes very difficult to find bugs. Professional programmers call this incremental development — build a little, test a little, repeat.
