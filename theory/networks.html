Networks & the Internet

How billions of devices communicate, share data, and connect the world — and what happens when you visit a website

The Idea

A World Connected

Right now, as you read this page, data is flying around the world at incredible speed. Your device sent a request across a network, through cables and routers, possibly across an ocean, to a server that sent this webpage back to you — all in a fraction of a second. How does that actually work?

A network is simply two or more devices connected together so they can communicate and share resources. Your home Wi-Fi is a network. Your school’s computers form a network. The internet is the biggest network of all — a global network of networks connecting billions of devices.

Understanding networks is essential because nearly every modern application relies on them: streaming music, sending messages, playing online games, backing up to the cloud, and even your smart fridge checking the weather.

In this topic you will learn:

  • The different types of networks (LAN, WAN, PAN)
  • Network topologies (how devices are arranged)
  • Protocols (the rules devices follow to communicate)
  • How data travels across networks (packet switching)
  • Network security and cyber threats

Types of Networks

LAN (Local Area Network)

A LAN covers a small geographical area, such as a single building, a school, or a home. All the hardware (cables, switches, routers) is typically owned by the organisation or individual.

  • Example: The computers in your school’s IT room connected to the school’s server
  • Fast data transfer speeds
  • Relatively cheap to set up and maintain

WAN (Wide Area Network)

A WAN covers a large geographical area, connecting LANs across cities, countries, or even continents. The infrastructure is usually owned by telecommunications companies.

  • Example: A company with offices in London and Edinburgh connected together
  • The internet is the largest WAN in the world
  • Slower than a LAN (data has to travel further), more expensive to maintain

PAN (Personal Area Network)

A PAN covers a very small area — typically within a few metres of a person. It connects personal devices.

  • Example: Your phone connected to your wireless earbuds via Bluetooth
  • Very short range, low power consumption

Network Topologies

A topology is the arrangement or layout of how devices are connected in a network.

Star Topology

Every device is connected to a central switch or hub. All data passes through this central device. Think of it like a spider’s web with the switch at the centre.

  • Advantages: If one cable fails, only that device is affected. Easy to add new devices. Fast performance because each device has its own connection.
  • Disadvantages: If the central switch fails, the entire network goes down. Requires more cabling than a bus topology. The central switch can be expensive.

Bus Topology

All devices are connected to a single main cable (called the backbone or bus). Data sent by one device travels along the cable in both directions, and all devices can see it.

  • Advantages: Cheap and easy to set up. Uses less cabling than a star topology.
  • Disadvantages: If the main cable fails, the whole network goes down. Performance slows as more devices are added (data collisions). Difficult to troubleshoot faults. Security issues as all devices can see all data.

Mesh Topology

Every device is connected to every other device (full mesh) or to several other devices (partial mesh). Data can take multiple paths between any two devices.

  • Advantages: Very reliable — if one connection fails, data can take an alternative route. No single point of failure. Fast communication.
  • Disadvantages: Very expensive (lots of cabling and network ports). Complex to set up and maintain. Mostly used for critical infrastructure like the internet’s backbone.
Key Concept: Star vs Bus vs Mesh Star is the most common in schools and offices (reliable, easy to manage). Bus is simple but fragile (one cable break = network down). Mesh is the most resilient but most expensive. In exams, you should be able to draw each topology and explain the pros and cons.

Network Protocols

A protocol is a set of rules that devices must follow to communicate with each other. Without protocols, devices would be like people speaking different languages — they could not understand each other.

ProtocolFull NamePurpose
TCP/IPTransmission Control Protocol / Internet ProtocolThe fundamental protocol of the internet. TCP ensures data is delivered reliably and in order. IP handles addressing and routing.
HTTPHyperText Transfer ProtocolUsed for loading web pages. Your browser sends an HTTP request; the server sends back the page.
HTTPSHTTP SecureSame as HTTP but with encryption, keeping data private (e.g. bank logins, online shopping). Look for the padlock icon in your browser.
FTPFile Transfer ProtocolUsed for uploading and downloading files between computers. Often used by web developers to upload website files to a server.
SMTPSimple Mail Transfer ProtocolUsed for sending emails from a client to a server, or between servers.
POP3Post Office Protocol 3Used for receiving emails. Downloads emails to your device and typically deletes them from the server.
IMAPInternet Message Access ProtocolUsed for receiving emails. Keeps emails on the server so you can access them from multiple devices (more modern than POP3).
DNSDomain Name SystemTranslates human-friendly domain names (e.g. google.com) into IP addresses (e.g. 142.250.187.46) that computers use to find each other.

The TCP/IP Layer Model

The TCP/IP model divides network communication into four layers. Each layer has a specific job and passes data to the layer above or below it. This layered approach means each layer can be developed and updated independently.

LayerNameWhat It DoesProtocols
4 (top)ApplicationProvides network services directly to the user/software (e.g. web browsing, email)HTTP, HTTPS, FTP, SMTP, DNS
3TransportBreaks data into packets, ensures reliable delivery, and reassembles at the other endTCP, UDP
2InternetHandles addressing (IP addresses) and routing packets across networksIP
1 (bottom)Network InterfaceHandles the physical transmission of data (electrical signals, Wi-Fi radio waves, etc.)Ethernet, Wi-Fi

Packet Switching

When you send data across the internet — an email, a photo, a web page request — it does not travel as one big block. Instead, it is split into small chunks called packets.

Each packet contains:

  • Header — The source IP address, destination IP address, packet number (so packets can be reassembled in order), and the total number of packets
  • Payload — A chunk of the actual data being sent
  • Trailer/footer — Error-checking data to verify the packet arrived correctly

How it works:

  1. The data is split into packets
  2. Each packet is sent independently — they may take different routes across the network
  3. Routers at each point decide the best path for each packet based on current network traffic
  4. At the destination, packets are reassembled in the correct order using the packet numbers
  5. If any packets are missing or corrupted, they are re-requested
Key Concept: Why Packet Switching? Packet switching is efficient because no single path is reserved for one communication — the network is shared. If one route is congested or fails, packets can take alternative routes. This makes the internet robust and able to handle millions of simultaneous users.

IP Addressing

Every device on a network needs a unique address so that data can be sent to the right place. This is the IP address (Internet Protocol address).

IPv4

  • Four numbers separated by dots, each between 0 and 255
  • Example: 192.168.1.1
  • Uses 32 bits, giving approximately 4.3 billion unique addresses
  • Running out of addresses due to the huge number of internet-connected devices

IPv6

  • Eight groups of four hexadecimal digits, separated by colons
  • Example: 2001:0db8:85a3:0000:0000:8a2e:0370:7334
  • Uses 128 bits, giving approximately 340 undecillion (3.4 × 1038) unique addresses
  • Created to solve the IPv4 shortage — enough addresses for every grain of sand on Earth, many times over

Network Security

Networks are incredibly useful, but they also create vulnerabilities. Here are the key security measures:

Firewalls

A firewall monitors and filters incoming and outgoing network traffic based on a set of security rules. It can block suspicious connections and prevent unauthorised access. Firewalls can be hardware devices or software programs.

Encryption

Encryption scrambles data into an unreadable format so that only authorised recipients can read it. There are two main types:

  • Symmetric encryption — The same key is used to encrypt and decrypt the data. Fast, but the key must be shared securely.
  • Asymmetric encryption — Uses a pair of keys: a public key (shared openly) to encrypt, and a private key (kept secret) to decrypt. More secure for internet communication (used in HTTPS).

Authentication

Verifying that a user or device is who they claim to be. Methods include passwords, biometrics (fingerprint, face recognition), two-factor authentication (2FA), and digital certificates.

MAC Address Filtering

Every network device has a unique MAC address (Media Access Control) burned into its hardware. A network can be configured to only allow devices with approved MAC addresses to connect. However, MAC addresses can be spoofed (faked), so this should not be the only security measure.

Network Threats

Malware

Malicious software designed to damage, disrupt, or gain unauthorised access to a system:

  • Virus — Attaches itself to a legitimate program and spreads when that program is run. Can corrupt or delete files.
  • Worm — Self-replicating malware that spreads across networks without needing a host program. Can consume bandwidth and crash networks.
  • Trojan — Disguises itself as legitimate software but performs malicious actions in the background (e.g. opening a “backdoor” for hackers).
  • Ransomware — Encrypts the victim’s files and demands payment (a ransom) for the decryption key. Has targeted hospitals, schools, and businesses.
  • Spyware — Secretly monitors user activity and collects data (passwords, browsing habits, keystrokes) without consent.

Other Attacks

  • Phishing — Fake emails or websites that trick users into revealing personal information (passwords, bank details). Often disguised as messages from trusted organisations.
  • Brute force attack — Systematically trying every possible password combination until the correct one is found. Strong, long passwords take much longer to crack.
  • DDoS (Distributed Denial of Service) — Flooding a server with so many requests that it cannot respond to legitimate users. Often uses a “botnet” (network of compromised computers).
  • SQL injection — Inserting malicious SQL code into a website’s input field (e.g. a login form) to manipulate the database. Can expose, modify, or delete data.
  • Social engineering — Manipulating people (rather than technology) into giving up confidential information. Includes phishing, pretexting (creating a fake scenario), and tailgating (following someone into a secure area).
Common Mistake: Students often confuse viruses and worms. A virus needs a host program to attach to and requires human action (like opening a file) to spread. A worm is standalone and spreads automatically across networks without any human action. Both are types of malware, but they work differently.

Q1: What is the difference between a LAN and a WAN?

Answer: A LAN (Local Area Network) covers a small geographical area such as a single building, school, or home, and the hardware is usually owned by the organisation. A WAN (Wide Area Network) covers a large geographical area (cities, countries, or the whole world) and typically uses infrastructure owned by telecommunications companies. The internet is the largest WAN.

Q2: In a star topology, what happens if the central switch fails?

Answer: If the central switch (or hub) fails in a star topology, the entire network goes down because all devices rely on the central switch to communicate. This is the main disadvantage of a star topology — the central switch is a single point of failure. However, if just one cable or device fails, only that device is affected and the rest of the network continues working.

Q3: What does DNS stand for, and what does it do?

Answer: DNS stands for Domain Name System. It translates human-friendly domain names (like google.com) into IP addresses (like 142.250.187.46) that computers use to locate each other on the network. DNS works like a phone book for the internet — you look up a name and get the number you need to make the connection.

Q4: Name the four layers of the TCP/IP model, from top to bottom.

Answer: From top to bottom: 1. Application layer (provides services to the user, e.g. HTTP, FTP, SMTP). 2. Transport layer (breaks data into packets, ensures reliable delivery, e.g. TCP). 3. Internet layer (handles IP addressing and routing). 4. Network Interface layer (handles physical transmission of data, e.g. Ethernet, Wi-Fi).

Q5: Explain how packet switching works. Why is data split into packets?

Answer: Data is split into small packets, each containing a header (source/destination addresses, packet number), payload (a chunk of the actual data), and error-checking information. Packets are sent independently and may take different routes across the network. At the destination, they are reassembled in the correct order. Data is split into packets because: (1) the network can be shared efficiently between many users, (2) if one route fails, packets can take alternative paths, and (3) lost or corrupted packets can be individually re-requested rather than resending everything.

Q6: Why was IPv6 created when we already had IPv4?

Answer: IPv4 uses 32-bit addresses, allowing approximately 4.3 billion unique addresses. With billions of internet-connected devices (phones, laptops, smart devices, servers), IPv4 addresses have run out. IPv6 uses 128-bit addresses, providing approximately 3.4 × 1038 addresses — essentially unlimited. IPv6 was created to ensure every device can have a unique address as the internet continues to grow.

Q7: What is the difference between symmetric and asymmetric encryption?

Answer: Symmetric encryption uses the same key for both encryption and decryption. It is fast but requires the key to be shared securely between both parties. Asymmetric encryption uses a pair of keys: a public key (shared openly) to encrypt the data, and a private key (kept secret) to decrypt it. Asymmetric encryption is more secure for internet communication because the private key never needs to be shared. HTTPS uses asymmetric encryption.

Q8: What is a DDoS attack, and how does it work?

Answer: A DDoS (Distributed Denial of Service) attack floods a server or network with an overwhelming number of requests from many different sources (often a botnet of compromised computers). The server becomes unable to handle legitimate traffic and either slows to a crawl or crashes entirely. The goal is to make a website or service unavailable, not to steal data.

Q9: Explain the difference between a virus and a worm.

Answer: A virus attaches itself to a legitimate program or file and only spreads when a user runs that infected program (requires human action). A worm is standalone malware that replicates and spreads automatically across networks without needing a host program or human action. Both can cause damage, but worms can spread much faster because they do not need anyone to open or run anything.

Q10: What is SQL injection, and how can it be prevented?

Answer: SQL injection is an attack where malicious SQL code is entered into an input field on a website (e.g. a login form or search box). If the website does not properly validate inputs, the SQL code can be executed on the database, potentially allowing the attacker to view, modify, or delete data. It can be prevented by: (1) input validation/sanitisation — checking and cleaning all user input before processing it, (2) parameterised queries — using prepared statements that separate SQL code from user data, and (3) limiting database permissions so the web application only has access to what it needs.

Staying Safe Online

Now that you understand how networks work and the threats that exist, here are some practical steps you can take to protect yourself:

  • Use strong, unique passwords — At least 12 characters, mixing uppercase, lowercase, numbers, and symbols. Use a password manager so you do not have to remember them all. Never reuse passwords across different accounts.
  • Enable two-factor authentication (2FA) — This adds a second layer of security (like a code sent to your phone) so even if someone gets your password, they still cannot access your account.
  • Look for HTTPS — Before entering any personal information on a website, check for the padlock icon and “https://” in the address bar. This means your connection is encrypted.
  • Be sceptical of emails and links — Phishing attacks are increasingly sophisticated. If an email asks you to click a link or enter personal information, verify it is genuine before acting. Check the sender’s actual email address, not just the display name.
  • Keep software updated — Updates often contain security patches that fix vulnerabilities. Enable automatic updates where possible.
  • Think before you share — Information you post online is very difficult to remove completely. Consider whether you would be happy for a teacher, parent, or future employer to see it.

The internet is an incredible tool for learning, creativity, and connection. Understanding how it works — and where the risks lie — empowers you to use it confidently and safely.
